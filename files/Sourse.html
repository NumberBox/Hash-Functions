Исследование хеш-функций


Хеш-функция - функция, осуществляющая преобразование массива входных данных произвольной длины в выходную битовую строку установленной длины, выполняемое определённым алгоритмом. Преобразование, производимое хеш-функцией, называется хешированием. Исходные данные называются «прообразом», а полученные «хешем».

В хешировании исходные данные преобразуются в хеш-код, при этом между входными и выходными значениями нет соответствия. Возвращаемые хеш-функцией строки менее разнообразны, чем значения входного массива. Случай, при котором хеш-функция преобразует более чем один массив входных данных в одинаковые строки, называется «коллизией». Вероятность возникновения коллизий используется для оценки качества хеш-функций.


Свойства хеш-функций:


1) Сжатие.

Хеш-функция h сжимает исходное сообщение (прообраз) x, получая при этом фиксированное значение y заданной длины.


2) Простота вычисления.

Для заданной функции h и сообщения x, h(x) вычисляется не выше, чем с полиноминальной сложностью.


3) Стойкость к вычислению прообраза.

Невозможность нахождения прообраза для любых хеш значений. Для заданной функции h невозможно найти прообраз x, при известном y = h(x) для любого y.


4) Стойкость к вычислению второго прообраза.

Невозможность вычисления второго прообраза, который давал бы тот же хеш, как и заданный. Для заданной функции h и прообраза x невозможно найти другой прообраз x' не равный x, при котором h(x') = h(x).


5) Стойкость к коллизиям.

У функции h не должно быть двух одинаковых значений y, для любых двух исходных прообразов x.


Условные обозначения используемые в программе:

x - числовое представление хешируемой строки, полученное путем сложения двоичных числел отдельных символов в кодировке IBM cp866,

M - размер ключевого пространства хеш-функции (количество всевозможных хеш-значений которое может принимать хеш),
